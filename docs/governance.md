# DAO 거버넌스 설계 — OTR Universe

> LORE 토큰 기반의 탈중앙화 세계관 거버넌스 시스템

---

## 거버넌스 구조 개요

```
                    커뮤니티 (LORE 토큰 보유자)
                            │
              ┌─────────────┼─────────────┐
              ↓             ↓             ↓
         제안 제출       투표 참여    위임(Delegation)
              │
              ↓
     ┌─────────────────────┐
     │  Lorekeeper Council │  ← 선출된 검토 위원회
     │  (Canon 충돌 검토)  │
     └─────────────────────┘
              │
              ↓
         공개 투표
              │
     ┌────────┴────────┐
     ↓                 ↓
  승인 → Canon 편입   부결 → Tier 4 보관 or 재제안
              │
     [창립팀 Core Veto 가능 — 연 3회 한도]
```

## 제안 유형 (LIP Types)

### Type A: Lore Addition (세계관 추가)
새로운 캐릭터, 장소, 사건, 세력을 Canon에 추가

| 항목 | Tier 1 | Tier 2 | Tier 3 |
|------|--------|--------|--------|
| 제안 최소 스테이킹 | 5,000 LORE | 1,000 LORE | 500 LORE |
| 토론 기간 | 7일 | 5일 | 3일 |
| 투표 기간 | 7일 | 5일 | 3일 |
| 승인 threshold | 75% | 60% | 50% |
| Quorum (최소 참여) | 15% | 10% | 5% |

---

### Type B: Canon Elevation (Tier 격상)
Tier 3 외전이 Tier 1~2 정식 Canon으로 편입 신청

| 항목 | → Tier 1 | → Tier 2 |
|------|----------|----------|
| 제안 최소 스테이킹 | 10,000 LORE | 3,000 LORE |
| 조건 | 해당 OTR 공개 후 30일 경과 | 해당 OTR 공개 후 14일 경과 |
| 토론 기간 | 10일 | 7일 |
| 투표 기간 | 7일 | 5일 |
| 승인 threshold | 75% | 65% |
| Quorum | 20% | 12% |

---

### Type C: Lore Amendment (설정 수정)
기존 Canon 설정을 수정하거나 보완

| 항목 | 소폭 수정 | 대폭 수정 |
|------|-----------|-----------|
| 판단 기준 | 기존 설정 유지, 세부 추가 | 기존 설정 의미 변경 |
| 제안 최소 스테이킹 | 2,000 LORE | 20,000 LORE |
| 승인 threshold | 60% | 80% |
| Quorum | 10% | 25% |

---

### Type D: Governance Rule Change (거버넌스 규칙 변경)
투표 파라미터, 토큰 정책 등 시스템 자체 변경

- 제안 최소 스테이킹: 50,000 LORE
- 토론 기간: 14일
- 투표 기간: 10일
- 승인 threshold: 80%
- Quorum: 30%

## 투표 메커니즘

### 기본 투표 방식
```
투표력 = 보유 LORE 토큰 수

예시:
- 1,000 LORE 보유 → 1,000표
- 10,000 LORE 보유 → 10,000표
```

### 고도화: 쿼드라틱 투표 (Phase 3 도입 예정)
고래(대량 보유자)의 독점 방지를 위해 Type A/B 투표에 적용

```
투표력 = √(보유 LORE 토큰 수)

예시:
- 1,000 LORE 보유 → 약 31.6표
- 10,000 LORE 보유 → 100표
- 100,000 LORE 보유 → 316표
```

> 쿼드라틱 투표는 고래 1명보다 소액 보유자 100명의 의견이 더 강하게 반영되도록 설계

### 위임(Delegation)
- LORE 토큰 직접 투표가 어려운 경우 신뢰하는 Lorekeeper에게 투표권 위임 가능
- 위임은 언제든 철회 가능
- 위임받은 Lorekeeper는 투표 전 자신의 입장을 공개 게시해야 함

## Lorekeeper Council

### 역할
- 새 제안의 기존 Canon 충돌 여부 검토 (필수)
- 세계관 일관성 가이드 작성 및 유지
- 분쟁 발생 시 1차 중재

### 선출 방식
- 분기별 1회 선출
- LORE 토큰 스테이킹으로 후보 등록 (10,000 LORE)
- 커뮤니티 투표로 5~9인 선출
- 임기: 1분기 (재선 가능, 최대 3연임)

### 인센티브
- 분기당 Treasury에서 LORE 토큰 보상
- 역할 수행 시 거버넌스 투표 가중치 1.5배

## 투표 진행 플로우

```
1. 제안 제출
   ├── LIP 문서 작성 및 Discord/플랫폼에 공개
   ├── 제안 스테이킹 LORE 잠금
   └── Lorekeeper Council에 검토 요청

2. 검토 기간 (Lorekeeper Review)
   ├── Canon 충돌 여부 판정
   ├── 충돌 없음 → 공개 토론 단계 진행
   └── 충돌 있음 → 제안자에게 수정 요청 or 공개 경고 후 진행

3. 공개 토론 (Discussion Period)
   ├── Discord #lore-proposals 채널에서 공개 토론
   ├── 제안자는 질문에 응답할 의무
   └── 제안 수정은 토론 종료 48시간 전까지만 가능

4. 투표 (Voting Period)
   ├── Snapshot or 온체인 투표 시작
   ├── 찬성 / 반대 / 기권 3가지 선택
   └── 투표는 공개 (누가 어떻게 투표했는지 볼 수 있음)

5. 결과 처리
   ├── 승인: Quorum 충족 + Threshold 달성
   │   └── Lore DB에 공식 등록 + 제안자 LORE 보상 지급
   ├── 부결: Threshold 미달
   │   └── 스테이킹 LORE 반환, Tier 4로 아카이브
   └── Quorum 미달: 재투표 1회 허용 (7일 후)
```

---

## 스테이킹 반환 및 패널티

| 결과 | 스테이킹 처리 |
|------|---------------|
| 제안 승인 | 전액 반환 + 보상 추가 |
| 제안 부결 | 전액 반환 |
| 제안 철회 (토론 전) | 전액 반환 |
| 제안 철회 (토론 중) | 50% 반환, 50% Treasury |
| 악의적 제안 (스팸, 충돌 고의) | 전액 소각 (커뮤니티 신고 → Lorekeeper 판정) |

---

## 긴급 거버넌스 (Emergency Governance)

세계관 외부 위협(저작권 침해, 심각한 설정 오류 확산 등) 발생 시:

- Lorekeeper Council 과반수 + 창립팀 동의로 즉시 발동
- 문제 설정 즉시 Tier 4로 격리
- 72시간 내 공개 투표로 최종 처리 결정

## Phase별 거버넌스 인프라

### Phase 1 (MVP)
- **플랫폼:** Discord 투표봇 + Snapshot (오프체인)
- **토큰:** 포인트 시스템 (추후 온체인 변환 가능)
- **투명성:** GitHub에 모든 제안서/투표 결과 커밋

### Phase 2
- **플랫폼:** 전용 웹 대시보드 구축
- **토큰:** ERC-20 LORE 토큰 발행 (Ethereum L2)
- **거버넌스:** Snapshot + Tally 연동

### Phase 3 (풀 DAO)
- **스마트 컨트랙트:** OpenZeppelin Governor 기반 온체인 거버넌스
- **자동화:** 투표 통과 시 Lore DB 자동 업데이트
- **Treasury:** 멀티시그 → DAO 자동 집행
